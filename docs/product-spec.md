# MooPing Loyalty Product Spec

ไฟล์นี้รวม concept, requirements, user flow, data model และ LINE OA plan ของ `MooPing_Loyalty` ไว้ที่เดียว เพื่อให้เอกสารโปรเจกต์ไม่แตกเป็นไฟล์เล็กเกินจำเป็น

## Overview

MooPing Loyalty เป็นระบบสะสมยอดซื้อสำหรับร้านหมูปิ้งที่ต้องการประสบการณ์หน้าร้านทันสมัยกว่าบัตรสะสมแต้มแบบกระดาษ

ปัญหาหลักคือร้านจำยอดสะสมลูกค้าได้ยาก โดยเฉพาะเมื่อลูกค้าซื้อหลายรอบ เช่น รอบแรก 7 ไม้ รอบต่อมา 3 ไม้ ระบบควรรวมเป็น 10 ไม้และให้ reward ทันที

ระบบนี้ช่วยให้พนักงานบันทึกยอดซื้อ คำนวณ reward อัตโนมัติ เปิดให้ลูกค้าเลือกของแถมเอง และเตรียมต่อยอด LINE OA notification ในอนาคต

## Core Rules

- ลูกค้าซื้อครบ 10 ไม้ ได้ reward 1 สิทธิ์
- ถ้าซื้อเกิน 10 ไม้ ยอดที่เหลือต้องสะสมต่อรอบถัดไป
- ลูกค้าเป็นคนเลือกของแถมเองเมื่อมี reward
- ระบบต้องเก็บยอดสะสมปัจจุบัน จำนวน reward ที่รอเลือก จำนวนสิทธิ์ที่เก็บไว้ และยอดซื้อรวม
- ถ้าลูกค้าเลือก `เก็บสิทธิ์ไว้ก่อน` สิทธิ์ต้องไม่หาย แต่ย้ายไปอยู่ใน `สิทธิ์ที่เก็บไว้`
- ยอดซื้อจากหน้า POS ต้องเป็นรายการรอยืนยันก่อน ไม่ควรบันทึกทันทีเมื่อกดปุ่มจำนวนไม้
- พนักงานต้องลบ 1 ไม้หรือล้างรายการรอยืนยันได้ก่อนบันทึกจริง
- หลังบันทึกจริงต้องยกเลิกรายการล่าสุดได้ เพื่อแก้กรณีกดผิดหน้าร้าน

## Reward Options

- หมูปิ้ง 1 ไม้
- ข้าวเหนียว 1 ห่อ
- เก็บสิทธิ์ไว้ก่อน

## Staff Flow

1. พนักงานเลือกลูกค้าจากหน้า POS
2. กดเพิ่มจำนวนไม้ที่ซื้อ เช่น `+3` หรือ `+10` เพื่อสร้างรายการรอยืนยัน
3. ถ้ากดผิด พนักงานกด `ลบ 1 ไม้` หรือ `ล้างรายการ`
4. ตรวจ preview ว่าหลังยืนยันจะได้ reward หรือไม่
5. กด `ยืนยันบันทึกยอด`
6. ระบบรวมยอดสะสมและคำนวณ reward
7. ถ้าครบ 10 ไม้ ระบบเพิ่ม reward ให้ลูกค้าทันที
8. ถ้ายืนยันผิด พนักงานกด `ยกเลิกรายการล่าสุด` เพื่อคืนยอดก่อนรายการนั้น

## Customer Flow

1. ลูกค้าดูสถานะบน iPad display
2. เมื่อลูกค้ามี reward ระบบแสดงตัวเลือกของแถม
3. ลูกค้าเลือกหมูปิ้ง ข้าวเหนียว หรือเก็บสิทธิ์ไว้ก่อน
4. ถ้าเลือกเก็บสิทธิ์ไว้ก่อน ระบบย้าย reward ไปที่สิทธิ์ที่เก็บไว้
5. ระบบบันทึกการเลือกและจำลอง LINE OA message

## Demo Flow

1. เปิดหน้า demo
2. เลือกลูกค้าตัวอย่าง
3. เพิ่มยอดซื้อจนใกล้หรือครบ 10 ไม้
4. เห็น reward เพิ่มขึ้น
5. เลือกของแถมหรือเก็บสิทธิ์ไว้ก่อน
6. เห็น mock LINE OA message

## Data Model

```ts
type Customer = {
  id: string;
  name: string;
  phone: string;
  sticks: number;
  pendingRewards: number;
  savedRewards: number;
  totalPurchased: number;
};

type RewardOption = {
  id: 'pork-stick' | 'sticky-rice' | 'save-later';
  name: string;
  icon: string;
};

type LineMessage = {
  id: number;
  time: string;
  text: string;
};
```

Future models:

- Purchase history
- Reward claim history
- Staff activity log
- LINE user profile

## LINE OA Plan

MVP ใช้ mock LINE OA panel ในหน้าเว็บก่อน เพื่อโชว์ concept และ user flow โดยยังไม่ต้องใช้ access token จริง

Production ต้องเชื่อม LINE Messaging API ผ่าน backend เท่านั้น เพราะ token ห้ามอยู่ใน frontend

Message events:

- ลูกค้าใกล้ครบ 10 ไม้
- ลูกค้าได้ reward ใหม่
- ลูกค้าเลือกของแถมแล้ว
- ลูกค้าเก็บสิทธิ์ไว้ก่อน
- ลูกค้าไม่ได้กลับมาซื้อนาน
- มีโปรโมชันพิเศษ

Example message:

```text
คุณสะสมหมูปิ้งแล้ว 8/10 ไม้ อีก 2 ไม้ก็เลือกของแถมได้แล้วครับ
```
