# 03 Loyalty Logic

กฎสำคัญคือทุก 10 ไม้เท่ากับ reward 1 สิทธิ์

ตัวอย่าง:

```text
ยอดเดิม 7 ไม้
ซื้อเพิ่ม 3 ไม้
รวม 10 ไม้
ได้ reward 1 สิทธิ์
ยอดสะสมรอบใหม่เหลือ 0 ไม้
```

ถ้าซื้อเกิน 10 ไม้ เช่น มี 8 ไม้แล้วซื้อเพิ่ม 5 ไม้:

```text
รวม 13 ไม้
ได้ reward 1 สิทธิ์
ยอดสะสมรอบใหม่เหลือ 3 ไม้
```

## POS Confirmation

หน้า POS ไม่ควรบันทึกยอดทันทีเมื่อพนักงานกดจำนวนไม้ เพราะตอนขายจริงมีโอกาสกดผิดได้

Flow ที่ปลอดภัยกว่า:

```text
กดจำนวนไม้
สร้างรายการรอยืนยัน
ตรวจ preview
ยืนยันบันทึกยอด
ถ้าผิดหลังยืนยัน ให้ undo รายการล่าสุด
```

แนวนี้ช่วยลดความเสี่ยงของยอดสะสมผิด และยังทำให้พนักงานแก้ไขเร็วโดยไม่ต้องเปิดระบบหลังบ้าน

## Saved Rewards

ถ้าลูกค้าเลือก `เก็บสิทธิ์ไว้ก่อน` ระบบต้องไม่ลบสิทธิ์ทิ้ง แต่เปลี่ยนจาก reward ที่รอเลือกไปเป็นสิทธิ์ที่เก็บไว้

```text
pendingRewards 1
ลูกค้าเลือกเก็บสิทธิ์ไว้ก่อน
pendingRewards 0
savedRewards 1
```

เมื่อกลับมาใช้ภายหลัง ลูกค้าควรเลือกของแถมจาก `savedRewards` ได้เหมือน reward ปกติ
