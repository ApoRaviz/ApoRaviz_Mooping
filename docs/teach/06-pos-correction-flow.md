# 06 POS Correction Flow

หน้าร้านจริงมีความเสี่ยงเรื่องกดผิดสูง เพราะพนักงานต้องกดเร็วระหว่างขาย

ดังนั้น POS flow ไม่ควรบันทึกยอดทันทีหลังแตะ `+1`, `+3`, `+5`, หรือ `+10`

## Safer Flow

```text
เลือกสินค้า / จำนวนไม้
สร้างรายการรอยืนยัน
ตรวจ preview
ยืนยันบันทึกยอด
ถ้าผิดหลังยืนยัน ใช้ undo latest sale
```

## Why It Matters

- ลดโอกาสสะสมแต้มผิด
- พนักงานแก้พลาดได้เร็ว
- ลูกค้าเห็นความโปร่งใสของยอดสะสม
- ระบบดูเหมือน product ที่พร้อมใช้หน้าร้าน ไม่ใช่แค่ demo
